# Knowledge

## 1. What this package is

**Knowledge** is a public, Obsidian-first knowledge atlas founded on the **Structured Unity Framework (SUF)**.

It is designed to be:

- a navigable map of major knowledge areas and subareas
- a graph-friendly study package rather than a linear handbook
- a sibling package to SUF rather than a replacement for it
- broad enough to move beyond SUF when the subject matter demands it
- explicit about where SUF is genuinely useful as scaffold, route logic, and relation discipline

Current snapshot:

- `v1.0.0` dated `2026-04-02`
- 9 cluster gateways
- 70 node notes
- typed relation vocabulary
- interconnected study routes

This is the first bounded public release of the Knowledge package inside the umbrella repository. It is stable and navigable, but it is not the main academic claim of the repository. It is included in the public release as supporting research infrastructure rather than as the principal academic argument.

## 2. How to use it

- Start with `_indexes/knowledge-index.md`.
- Use `_indexes/node-index.md` when you want the entire 70-node program in one place.
- Use `map/` for cluster gateways and domain notes.
- Use `study-routes/` for guided traversals through the graph.
- Use `_relations/` to keep cross-links typed and legible.
- Read `suf-relationship.md` for the clearest statement of how this project uses SUF.
- Read [Studying and Teaching with SUF](studying-and-teaching-with-suf.md) if you want a pedagogical entry point for courses, self-study, or introductory philosophy/science use.
- Read [Using This Research with Human Assistants and Reasoning Agents](../using-this-research-with-human-assistants-and-reasoning-agents.md) if you want guidance for collaborative navigation, accessibility support, or agent-assisted use.

## 3. Relationship to SUF

SUF contributes three things especially well here:

- a broad layered scaffold for moving across knowledge clusters
- relation discipline for linking notes without flattening them
- route logic for tracing how foundational, formal, applied, and reflexive areas connect

Knowledge is broader than SUF. Many notes should eventually use domain-native frameworks, competing systems traditions, or non-SUF literatures more heavily than SUF itself. The point is not to pretend SUF explains everything; the point is to let SUF organize a coherent, honest starting atlas where it helps.

For the framework-first side of that relationship, start in the sibling package:

- [Structured Unity Framework](../structured-unity-framework/README.md)

These are relative links inside the same research repository. If the projects are split into separate remotes later, replace them with hosted URLs.

## 4. Recommended Obsidian setup

This package is structured to work especially well with:

- **Graph view** for the full cluster, node, and route network
- **Local Graph** when you want a bounded view around one node or route
- **Tags** for grouping by note kind, cluster, route role, and status
- **Backlinks** for seeing how routes and nodes support each other
- **Outgoing Links** for spotting missing or weak cross-cluster connections
- **Properties** for filtering routes, deepened hubs, and SUF-related notes
- **Templates** when extending the atlas with new notes or routes

Markdown body links remain the main graph backbone. Properties add filtering and structure, but they are not a substitute for explicit links in the note body.

## 5. Package shape

- `_indexes/` = entry and flattened navigation surfaces
- `map/` = cluster gateways and node notes
- `study-routes/` = connected guided traversals
- `_relations/` = typed relation vocabulary
- `_templates/` = note and route templates
- `studying-and-teaching-with-suf.md` = pedagogical guide for self-study and teaching use

## 6. Working posture

Knowledge should stay:

- public and readable
- graph-native in Obsidian
- enthusiastic about SUF where SUF genuinely earns it
- honest about where SUF should recede behind better domain-native tools

That balance is part of the point of the project.
