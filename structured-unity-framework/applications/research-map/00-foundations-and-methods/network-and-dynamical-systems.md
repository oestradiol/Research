# Network and Dynamical Systems

## Role in the applications atlas

This node supplies the main formal toolkit for time-sensitive dependency analysis, alignment persistence, and lag structure.

## Why SUF applies here

SUF needs a way to describe interaction structure through time. Temporal networks and dynamical-systems language provide that without forcing all routes into one measurement protocol.

## Main objects of study

- temporal graphs and changing edge structure
- phase transitions and regime shifts
- synchronization, coupling, and instability
- persistence, intermittency, and lagged propagation

## Typical units and unitization problems

Units may be nodes, events, windows, or trajectories. The main issue is choosing units that preserve process structure rather than flattening everything into static aggregates.

## Typical scales

- micro: event-to-event relations
- meso: moving windows and cluster summaries
- macro: whole-system trajectories across bounded intervals

## Likely I / C / L / U relevance

This node is especially strong for `I`, `C`, and `L`. It supports `U` indirectly by clarifying when organized system-level signatures persist across windows and scales.

## One concrete SUF research question

What temporal-network summaries best capture the change from acute lockdown coordination to lower-acuity de-escalation in the New Zealand route?

## One bounded future route

A technical appendix comparing static dependency summaries with windowed temporal-network summaries for the same coded event ledger.
