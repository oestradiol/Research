# Taiwan Comparator Design

## Purpose

This file opens the first comparative branch after the New Zealand route cleared the current durability and ledger gate.

The Taiwan tranche remains bounded. It is now denser than the original starter pass, but it is still not a claim of full NZ-Taiwan symmetry.

## Why Taiwan first

Taiwan is the first comparator because the public official record makes three things visible early:

- command-centre activation
- border and transit restriction timing
- quarantine support, follow-up, and compliance language

That is enough for a bounded first comparative branch.

## Gate condition

The New Zealand baseline cleared the current gate before this comparator was opened:

- `38` coded NZ events total
- `29` coded NZ events in the main perturbation interval
- multiple durable implementation-side additions beyond the earlier `25`-event state
- late-transition authority-routing and review papers now admitted under the same archive rule
- all accepted additions admitted only with live plus fixed-archive support

## Comparator boundary

This comparator covers Taiwan's official acute command, border, quarantine-support, and quarantine-compliance coordination visible in archive-clean Taiwan CDC / CECC English materials from `2020-01-20` to `2020-04-23`.

Included by default:

- CECC activation and cross-agency command-centre language
- border-entry and transit restriction events
- home-quarantine support and care routing
- quarantine follow-up, local support-centre, and sanction language
- public-information guidance attached to those moves

Excluded by default:

- Taiwan's full later de-escalation path
- broader economic and education policy
- full tracing-system detail
- non-official secondary reconstructions

## Matched-window logic

The Taiwan comparator window is **functionally matched**, not date-matched, to the New Zealand route.

Reason:

- Taiwan's command-centre activation and border tightening began earlier
- Taiwan did not reproduce New Zealand's exact alert-level topology
- using the same calendar window would hide the actual coordination form visible in Taiwan's official record

Current Taiwan comparator window:

- `2020-01-20` to `2020-04-23`

This is the first acute border/quarantine coordination arc visible in the archive-clean official seed.

## Functional unit mapping

Use the New Zealand `sigma1` vocabulary as a functional translation layer, not as a claim of institutional identity.

| SUF `sigma1` label | Taiwan comparator mapping rule |
|---|---|
| `strategic executive coordination` | CECC command-centre decisions and explicit Executive Yuan cross-agency support where visible |
| `public-health policy and command` | Taiwan CDC disease-control, surveillance, medical-routing, and health-bureau coordination functions |
| `emergency-management coordination` | kept latent unless a distinct emergency-management layer is visible in official comparator material |
| `public-service system coordination` | local district offices, county/city support centres, and administrative support routing |
| `border-control coordination` | airport/port quarantine and entry-control operations |
| `enforcement and compliance` | quarantine enforcement, fines, group quarantine, and compliance follow-up |
| `public-information coordination` | CECC/Taiwan CDC public guidance, risk communication, and official public-facing updates |

## `sigma2` cluster mapping

- strategic coordination: `strategic executive coordination`, `public-service system coordination`
- response operations: `public-health policy and command`, `emergency-management coordination`, `border-control coordination`, `enforcement and compliance`
- public alignment: `public-information coordination`

## Source-family priority

Priority for the Taiwan comparator tranche:

1. official command and architecture surfaces
2. official border and quarantine implementation surfaces
3. official continuity and extension surfaces
4. retrospective official summaries only when they clarify boundary and unit logic

## Source-admission rule

No Taiwan source enters the current comparator tranche unless:

- the live public source is verified
- an exact fixed archive URL is verified
- the source materially adds boundary, timing, implementation, or dependency detail

Wildcard archive lookups are not accepted.
