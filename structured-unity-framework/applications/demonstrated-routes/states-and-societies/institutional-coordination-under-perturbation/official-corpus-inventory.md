# Official Corpus Inventory

## Purpose

This file is the first public assembly dossier for the New Zealand route's official source base.

It does not claim that the full corpus is finished. It records what is already assembled at the public-package level, what each source family currently does for the route, and what still needs to be added before the corpus can be treated as complete.

## Current posture

Current status: `first-pass public inventory`

What is already true:

- the core official and oversight source base is now assembled strongly enough to support a first-pass event ledger
- Families `A`, `B`, and `F` have a usable public core
- Family `C` now has both official-review chronology and direct event-page support
- Families `D` and `E` are now materially deeper, with a durable DPMC/Police implementation-side layer admitted under the live-plus-fixed-archive rule
- Family `G` remains intentionally deferred until the official base is fuller

## What the assembled public core already supports

The current public core is already strong enough to support:

- a bounded seven-unit `sigma1` architecture
- the locked `sigma2` and `sigma3` readings
- the main interval and both comparator windows
- a first-pass event ledger built from clear official coordination shifts and direct event pages
- a first conjoint comparison across Comparator A, the main perturbation interval, and Comparator B using the same public event family
- a public distinction between what is architecturally grounded now and what still needs deeper event-level sourcing
- a stronger direct source base for the current implementation-chain and sensitivity pass in D-family `D` and `E`, now filtered through an explicit live-plus-fixed-archive admission rule
