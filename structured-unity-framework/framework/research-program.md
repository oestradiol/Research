# Research Program

## 1. Purpose

This document is the public methods layer for Structured Unity Framework. It states how the framework can be studied responsibly: bounded domains, boundary rules, unitization, temporal windows, proxy bundles, source families, comparators, nulls, and robustness checks.

It does not replace the theory stack, prove phenomenology from metrics, or claim that one case confirms the whole framework.

## 2. Current bounded-release posture

## 3. First active route

### Domain strategy

**States and societies** come first because institutional coordination under perturbation is observable enough to support bounded research design without pretending to cover all social life.

### Subdomain strategy

**Institutional coordination under perturbation** is the first active route because it keeps timing central, makes interdependence legible, and supports explicit cross-scale reading.

### Current demonstrated case

**New Zealand pandemic coordination**, first acute national response cycle.

- main perturbation interval: `2020-02-28` to `2020-06-08`
- Comparator A: `2019-11-01` to `2020-01-31`
- Comparator B: `2020-05-14` to `2020-06-07`

Comparator B is locked as the lower-acuity de-escalation window. It follows the official chronology used in the Office of the Auditor-General and Royal Commission source base and should move by at most one day only if event-ledger assembly exposes a source-level timestamp discrepancy.

## 4. Boundary and unitization discipline

### Boundary

The route covers New Zealand's **core all-of-government pandemic coordination apparatus** during the first acute response cycle.

Included by default:

- central executive coordination
- public-health command and policy core
- emergency-management coordination
- public-service system coordination
- border-control coordination
- enforcement and compliance
- official public-information coordination

Excluded by default:

- international actors outside the core apparatus
- diffuse public sentiment as a total field
- media discourse as a whole ecology
- downstream actors included only for symbolic prominence

### Unitization

Units lie where institutional identity and operational coordination role intersect.

#### `sigma1` units

1. strategic executive coordination
2. public-health policy and command
3. emergency-management coordination
4. public-service system coordination
5. border-control coordination
6. enforcement and compliance
7. public-information coordination

#### `sigma2` clusters

- strategic coordination
- response operations
- public alignment

#### `sigma3`

- the bounded whole-system readout

### Boundary stress-test questions

1. Does inclusion or exclusion materially change dependency structure?
2. Does it materially change coordination timing?
3. Does it materially change implementation pathways?
4. Does it improve observability without creating unmanageable noise?
5. Does it preserve interpretability across `sigma1`, `sigma2`, and `sigma3`?

If the answer is consistently no, the actor stays outside the default boundary.

## 5. Primary measurement bundle

The current public bundle is now locked at the family level.

### `I` - integration

Use weighted coordination-dependency graph summaries centered on:

- dependency density
- cross-cluster connectivity
- participation breadth

### `C` - coherence

Use rolling alignment-persistence summaries centered on:

- directive consistency
- stability of coordinated sequencing
- cross-unit convergence

### `L` - latency

Use decision-to-implementation lag summaries centered on:

- median lag by unit and by chain
- dispersion across units and chains
- lag changes across Comparator A, the main perturbation interval, and Comparator B

### `U` - unity-like organization

Use a dashboard-style conjoint signature, not a single scalar. Interpret `U` comparatively across windows and scales and never as a universal score.

### Anti-fake-precision rule

This document locks the bundle, not one final estimator for every route. It does not freeze one threshold system, one weighting scheme, or one aggregation formula across all future domains.

## 6. Coding schema and source-family rule

### Event schema

Every coded event in the New Zealand route should include:

- `event_id`
- `timestamp_or_date`
- `issuing_unit`
- `receiving_units`
- `action_type`
- `dependency_type`
- `implementation_marker`
- `public_information_marker`
- `source_citation`
- `confidence_note`
- `scale_tag`

### Required source families

- `D-family A`: governance and coordination architecture
- `D-family B`: pandemic planning and public-health command material
- `D-family C`: official chronology and alert-level transitions
- `D-family D`: public-service and emergency-management coordination material
- `D-family E`: border, enforcement, and public-information records
- `D-family F`: oversight, audit, and retrospective review
- `D-family G`: academic retrospective and comparative literature, used only after Families A-F are assembled

### Priority rule

Official and oversight materials ground boundary, timing, and unit logic first. Academic retrospective work deepens interpretation later; it does not replace structural grounding.

## 8. Comparator and null logic

### Comparator A

Same-system ordinary-governance baseline: `2019-11-01` to `2020-01-31`.

### Comparator B

Lower-acuity contrast within the same response cycle: `2020-05-14` to `2020-06-07`.

### Structural null

Dependency shuffle or relation-preserving size null.

### Temporal null

Sequence shuffle null.

### Sensitivity checks kept active

- boundary perturbation
- cluster reassignment
- lag randomization
- coding-rule perturbation
- unit merge and split checks
- missingness and chronology-gap checks

## 9. Hypothesis family and weakening conditions

### Hypothesis family

- `H1`: stronger effective integration should support stronger unity-like organization
- `H2`: stronger coherence should support greater persistence of coordinated organization
- `H3`: latency should modulate when and how unity-like organization appears
- `H4`: different windows and scales should reveal different forms of unity-like organization
- `H5`: perturbation should make dependency, coherence, and lag structure more visible than ordinary governance

### Weakening conditions

The current framing weakens if these patterns recur:

- `I` and `C` rise without any sensible `U` improvement
- low-coherence intervals show indistinguishable durable `U` signatures
- latency adds no explanatory value after scale adjustment
- small unitization changes radically alter interpretation
- nulls match or outperform the structured `I` / `C` / `L` reading

## 10. Comparative extension posture

The first bounded comparative shortlist is now explicit.

### Taiwan

Recommended first comparator because it offers:

- stronger SARS-era preparedness
- earlier command-centre activation
- useful contrast on border management, tracing, and coordination speed

This is now the first active comparative branch because the New Zealand route has cleared the current durability and ledger gate.

### Australia

Recommended federal contrast because it offers:

- intergovernmental coordination complexity through National Cabinet and AHPPC
- a different implementation topology from New Zealand's more centralized apparatus
- an official inquiry record that supports bounded route design

### Comparative rule

Comparators must preserve functional equivalence, not just country-level similarity. Comparison should stay bounded to matched windows, matched route boundaries, and justified unit mappings.

## 14. Non-claims

This document does not claim:

- universal thresholds for unity-like organization across all domains
- strong consciousness attributions to states or institutions

- that operational summaries are identical with phenomenological givenness
