# Reference Guide

## 1. Why this directory now has four surfaces

The public package now separates:

- `source-registry.md` for canonical source metadata and durable public links
- `citation-and-archival-policy.md` for citation shape and archive rules
- `minimal-reading-list.md` for first orientation
- `standard-reading-list.md` and `extended-reading-list.md` for deeper, role-specific reading

This keeps the package readable without letting citation shape drift across files.

## 2. Reading-list roles

- **Minimal** (`minimal-reading-list.md`) - first-pass orientation across the framework, methods, route, and frontier
- **Standard** (`standard-reading-list.md`) - the active public anchor set for research-ready use
- **Extended** (`extended-reading-list.md`) - targeted depth for specialist objections, bridge disputes, or the AI frontier

## 3. Registry and policy roles

- `source-registry.md` is the canonical source layer for repeated public references
- `citation-and-archival-policy.md` states when to use DOI, when to use original-source-plus-archive, and how public files should cite without improvising

## 4. How to use this directory

Start with **minimal** if you are orienting.

Use **standard** if you are writing, reviewing, or pressure-testing the current public package.

Use **extended** only when a real objection or audience need calls for more depth.

Use the **source registry** when you need the exact public link, date, and scope note for a cited source.

## 5. What references do not do

References do not prove SUF by authority, replace argument, or create false ancestry. The routing discipline still lives in `../framework/literature-guide.md`.
