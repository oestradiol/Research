# Risk Register

## 1. Why this file exists

Documentation packages drift: scope inflates; README becomes a master theory; citations replace argument; applications stand in for universality; or release discipline degrades. This register names the main public-maintenance risks explicitly.

## 2. Core translation and architecture risks

- **Overclaim** in public wording.
- **Collapse** of layers into one doctrine.
- **Publication-pressure drift** — packaging or audience pressure distorts core architecture prematurely.
- **Literature laundering** — citations overstate what was established.

## 3. Role drift

Documents begin absorbing functions that belong elsewhere (methods into theory, theory into applications, and so on).

## 4. Pilot inflation

The New Zealand pilot is treated as if it **already proves** the whole framework.

**Mitigation:** bounded-route language in `applications/` and `framework/research-program.md`.

## 5. README inflation risk

The entry file grows until it **duplicates** the whole stack.

**Mitigation:** keep `README.md` an **orientation gateway**; depth lives in `framework/`.

## 6. Application inflation risk

The first route is mistaken for **universal proof**.

**Mitigation:** repeat **non-proof** and **boundedness** language in application files.

## 7. Category drift risk

Phenomenology, operations, and metaphysics **merge** undifferentiated.

**Mitigation:** claim types + `framework/framework-interface.md`.

## 8. Ontological hardening risk

Modeling language is read as **literal** final ontology.

**Mitigation:** `meta/terminology-policy.md` + interpretive tagging.

## 9. Framework collapse

The layered stack is flattened into a **single** doctrine.

## 10. Status-surface drift

Current-state, future-facing, and open-item files begin duplicating or blurring one another.

**Mitigation:** keep `docs/project-status.md`, `ROADMAP.md`, and `docs/pending-inventory.md` in their designated roles.

## 11. Public overclaim

The public package sounds like **accomplished universality** rather than domain-agnostic **extensibility**.

## 12. Domain collapse

The states/societies route is mistaken for the **whole project** rather than a bounded first application.

## 13. Premature public hardening

First-pass public decisions are treated as **immutable** before drafting pressure and feedback have tested them.

## 14. Infrastructure drift

Public package execution silently deviates from locked **release**, **reference**, or **versioning** rules.

**Mitigation:** `CONTRIBUTING.md`, `CHANGELOG.md`, `ROADMAP.md`; verify before releases.

## 15. Maintenance note

Review this register when **major** edits land or when **public** messaging changes. Patch releases: wording and links. **Structural** claim changes: document in `CHANGELOG.md` with scope discipline.
