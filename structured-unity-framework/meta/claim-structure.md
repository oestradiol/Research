# Claim Structure

## 1. Why claim typing matters

Mixed claim types (epistemic, structural, operational, interpretive, speculative) cause **silent upgrades**—readers treat models as proof or phenomenology as metrics. Typed claims are a core **anti-confusion** device.

## 2. Claim families used in the package

| Family | Typical content |
|--------|-----------------|
| Epistemic | Givenness, observation, bracketing |
| Structural | Patterns, relations, models as described |
| Operational | Measurement, comparison, tests |
| Interpretive | Meaning-rich extensions not secured as evidence |
| Speculative | Conceivable but not justified at stated level |

## 3. Examples of acceptable claim escalation

- From **structural** description to **operational** proposal when methods explicitly introduce indicators and tests.  
- From **operational** results to **interpretive** discussion when labeled as interpretation.

## 4. Examples of unacceptable claim inflation

- Treating a **proxy** as **identity** with a construct.  
- Presenting **interpretive** language as **measurement** results.  
- Using **one case** as proof of **universal** framework truth.

## 5. Relation to Framework Interface

`framework/framework-interface.md` is authoritative for how claim types interact with weak naturalization and layer separation. This file makes the discipline **explicit for contributors**.
