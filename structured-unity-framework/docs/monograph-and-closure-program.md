# Monograph and Closure Program

## Purpose

This file is the canonical long-range program for the next deliberate `v1.x` reopening after the active core was completed on `main`.

It defines the finish line, phase order, required artifacts, default comparator sequence, and acceptance gates for moving from the current bounded public package toward:

- a finished empirical monograph in `New Zealand core + comparator chapters` form
- strong multi-case closure within pandemic-governance inside the current states-and-societies route family
- a strong final empirical result in bounded form, without claiming universal validation or cross-domain closure

Use this file when planning later execution. Use [project-status.md](project-status.md) for current package posture, [pending-inventory.md](pending-inventory.md) for queued later work, and [../ROADMAP.md](../ROADMAP.md) for the compressed future-facing summary.

## Current phase state

- Phase 0 completed in the hosted `v1.0.1` baseline.
- Phase 1 completed in the hosted `v1.1.0` release.
- The next live phase is `v1.2` Taiwan completion.

## 1. Finish line

The long-range finish line is a **monograph-grade repo**, not merely a denser demonstrated route and not necessarily a journal-style manuscript package.

That means the public repo should eventually contain:

- a chapter-readable monograph structure
- a New Zealand monograph core that can stand as the primary empirical anchor
- a chapter-ready Taiwan route
- a bounded Australia federal comparator
- a bounded pandemic-governance closure note that states what travels across those cases and what does not
- a rival-framework positioning note that shows where SUF adds analytical value and where it remains bounded
- validation support for every stable public metric, table, and comparison surface that the monograph-facing docs depend on

The bounded closure target is **family-level**, not universal:

- `allowed`: pandemic-governance closure inside the current states-and-societies family
- `not allowed`: universal validation, cross-domain closure, field-settling theory claims

## 2. Locked defaults and non-goals

These defaults stay in force unless the methods docs are revised first and the route docs are updated in the same tranche:

- keep the current event schema as the public route contract
- keep the live-plus-fixed-archive source-admission rule
- keep the current window logic unless stronger source evidence forces an explicit revision
- keep the route-local estimator posture rather than introducing a universal scalar
- keep docs canonical and tools read-only

Locked planning defaults:

- monograph shape = New Zealand core plus comparator chapters
- comparator order = Taiwan, then Australia, then later optional comparator only if explicitly reopened
- finish line = monograph-grade repo rather than submission-form manuscript by default
- versioning = keep package metadata at `1.1.0` until a later hosted tag is actually cut; use `Unreleased` changelog sections meanwhile

Explicit non-goals:

- universal validation
- cross-domain closure
- field-settling theory posture
- automated public-doc rewriting
- direct tooling authorship over canonical research prose

## 4. Phase ladder

### Phase 0: Lock the program into the repo

Goal:

- make the long-range program discoverable and stable before later empirical reopening begins

Required outputs:

- this canonical program doc
- the monograph table of contents
- the monograph evidence map
- the bounded pandemic closure note scaffold
- the rival-framework positioning note scaffold
- synchronized references from roadmap, pending inventory, research program, comparative shortlist, tooling backlog, and index surfaces

Exit gate:

- every future-facing surface points to the same canonical long-range program
- future agents can find the plan from status, methods, comparative, navigation, and tooling docs without guessing

### Phase 1: `v1.1` New Zealand monograph hardening

Goal:

- reopen New Zealand as the monograph core and push it from active-core completion to monograph-grade saturation

Required work:

- exhaust the official and oversight discovery queue inside the locked New Zealand boundary and windows
- force every high-priority candidate into `promoted`, `rejected`, or `held with durable reason`
- deepen the New Zealand ledger until the unresolved queue is low-priority rather than structurally important
- upgrade current first-pass surfaces into stable monograph-baseline interpretation surfaces where the evidence now supports it
- produce chapter-ready New Zealand evidence mapping across architecture, escalation, implementation thickening, late-transition authority and review, de-escalation, estimator, nulls, and weakening conditions
- plan every stable monograph-facing New Zealand table and figure in canonical docs before any analysis-ready export work begins

Exit gate:

- New Zealand can stand as a monograph core chapter set, not only a demonstrated route
- no unresolved high-priority source or event gap remains inside the locked New Zealand windows
- every public New Zealand metric and table is reproducible from canonical Markdown artifacts
- the locked payoff sentence still survives the denser New Zealand evidence, or it is explicitly revised across every dependent surface in one tranche

### Phase 2: `v1.2` Taiwan completion

Goal:

- push Taiwan from bounded comparator to chapter-ready bounded route

Required work:

- deepen the Taiwan corpus beyond the current acute tranche under the same archive rule
- expand the Taiwan ledger until the branch can support a standalone bounded empirical chapter rather than only a comparison note
- improve functional separation across executive, health, border, local-administrative, logistics, compliance, and public-alignment functions where the source base actually supports it
- search explicitly for a denser Taiwan lag surface and publish it only if the evidence is clean enough
- add a Taiwan evidence map aligned to the monograph chapter structure

Exit gate:

- Taiwan can sustain a standalone bounded chapter
- the branch no longer depends on starter-language or implied future completion
- Taiwan either has a defensible lag subsection or an explicit lag-limit section
- every stable Taiwan metric and table validates cleanly

### Phase 3: `v1.3` Australia federal comparator

Goal:

- open Australia as the second comparator in Commonwealth-focused federal form

Required work:

- lock the Australian route boundary, unit mappings, source-family rule, windows, and null logic before broad ledger growth
- build around National Cabinet, AHPPC, Commonwealth border and quarantine coordination, public-health command, and official inquiry material
- include only the minimum state-interface layer needed for dependency chains
- keep the route bounded enough that federal complexity does not destroy comparability
- produce a parallel bounded route package with design, corpus, discovery, ledger, estimator, comparison, and limits surfaces

Exit gate:

- Australia exists as a real bounded comparator rather than a placeholder candidate
- federal boundary and unit-mapping rules are explicit enough for reproducible comparison
- the route yields at least one clear implementation-side chain plus enough evidence for a comparator chapter
- the three-case comparison can now be built without inventing new boundary logic ad hoc

### Phase 4: `v1.4` bounded pandemic-governance closure and monograph-grade repo

Goal:

- turn the repo into a monograph-grade public research object with bounded three-case closure

Required work:

- build the synthesis chapter set across New Zealand, Taiwan, and Australia
- state exactly what patterns travel across the three cases and what patterns do not
- lock the closure claim only at the pandemic-governance family level
- add a rival-framework positioning note showing what SUF explains better than executive-command-only, communication-as-downstream, or loose crisis-governance summaries
- update claims, payoff, status, and monograph-facing docs together so claim strength matches the multi-case evidence

Exit gate:

- New Zealand, Taiwan, and Australia together support a strong bounded empirical result
- the repo can be read as a finished monograph-grade package even without journal-style manuscript formatting
- the closure claim is strong inside the pandemic-governance family and explicitly non-universal outside it
- rival-framework pressure is answered in a bounded, source-backed way rather than left implicit

### Phase 5: `v1.5` analysis-ready tooling

Goal:

- add stronger export and validation support only after the empirical surfaces are stable

Required work:

- add structured exports derived from canonical Markdown
- add deterministic generation or validation for monograph tables, figure inputs, comparator summaries, and evidence-map integrity
- add cross-case validators for New Zealand, Taiwan, Australia, and closure-note claims wherever those claims depend on stable metrics
- keep docs canonical and tools read-only

Exit gate:

- every stable monograph-facing public metric can be validated from canonical docs
- cross-case tables and summary outputs are reproducible
- tooling docs match actual capabilities and no tool rewrites public prose

### Phase 6: `v1.6` research-ops tooling

Goal:

- add maintenance support only after the monograph-grade repo exists and actual upkeep pain justifies it

Required work:

- add backlog hygiene and discovery-state integrity support
- add artifact-completeness checks and future-agent handoff support
- keep any workflow helpers subordinate to public-doc authority and explicit human review

Exit gate:

- maintenance tooling reduces drift without changing authorship
- future agents can pick up long-range work from explicit queue state rather than inference

## 5. Validation and acceptance

After every later phase, run:

- `research-tools validate all`
- `nix develop -c python -m pytest`
- `nix develop -c ruff check .`
- changelog and version consistency checks
- human review of the release-readiness report

Acceptance scenarios:

- `NZ monograph core`
  all high-priority New Zealand source candidates are resolved and the route can be read as a chapter-ready empirical core
- `Taiwan chapter readiness`
  Taiwan supports standalone bounded interpretation and no longer depends on starter framing
- `Australia comparator readiness`
  the federal comparator is bounded, reproducible, and not swamped by uncontrolled state-level scope
- `Bounded closure`
  the three-case synthesis states exactly what is supported at the pandemic-governance family level, with explicit weakening conditions
- `Monograph-grade repo`
  chapter map, evidence map, methods, comparative closure, rival-framework note, tables and figures, and appendices all exist and cross-reference cleanly
- `Tooling alignment`
  every stable public metric or table used in monograph-facing docs is validated by the read-only tooling layer
