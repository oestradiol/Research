# Framework Overview

## 1. Executive overview

Structured Unity Framework presents a layered way to think about unity-like organization: when a structured system behaves as more than a loose aggregate at a given temporal window and structural scale. The stack separates phenomenological grounding, structural and generative modeling, dynamic variables, and explicit bridge rules so phenomenology, science, and interpretation do not collapse into one doctrine.

## 2. The four theory layers

1. **Structural Phenomenology** - awareness-first epistemic ground
2. **Informational Awareness Framework** - structural and generative modeling stance, not a finalized ontology
3. **Unity Dynamics Framework** - `I`, `C`, `L`, and `U` at explicit `(tau, sigma)`
4. **Framework Interface** - claim types, weak naturalization, non-circularity, and bridge discipline

## 3. Why the stack is split

- different layers answer different questions
- merging them invites category drift and overclaim
- methods and literature sit beside the theory core rather than replacing it

## 4. How theory relates to the research program

`framework/research-program.md` now states a research-ready provisional design rather than only a placeholder methods posture. It locks the current New Zealand route architecture, source-family rule, comparator logic, and primary measurement bundle while still avoiding fake precision.

`applications/demonstrated-routes/states-and-societies/institutional-coordination-under-perturbation/estimator-implementation.md` now adds the first route-local implementation layer for `I`, `C`, `L`, and dashboard-style `U`.

## 5. How the literature guide fits in

`framework/literature-guide.md` is routing-oriented. It tells readers which source families support which moves and which moves those sources do **not** justify.

`references/source-registry.md` is the canonical metadata layer underneath that routing. `references/citation-and-archival-policy.md` locks the DOI-first / original-plus-archive rule for public citations.

## 6. How applications fit in

`applications/` now has two coordinated parts:

- `demonstrated-routes/` for real bounded uses of SUF
- `research-map/` for the targeted public atlas of next domains and comparative directions

The current demonstrated route is the New Zealand pandemic-coordination package under the states-and-societies family.

## 7. What remains open

- fuller corpus completeness for the New Zealand route
- further estimator refinement and robustness pressure
- later comparative execution beyond the current Taiwan comparator baseline
- later analysis-ready or research-ops tooling beyond the current read-only validation layer

These open items do not erase the current bounded `v1.0.1` package; they mark the next empirical and validation layers of work on `main`.
