# Contribution and Payoff Note

## Purpose

This file drafts the current public contribution/payoff candidates without pretending one final sentence is already locked.

The current evidence base is the `30`-event New Zealand route with first-pass `I`, `C`, `L`, conjoint window comparison, and bounded sensitivity/null work.

## Candidate 1

### Candidate statement

SUF shows that the New Zealand response should not be read only as centralized executive command, because public-information coordination is structurally central to the working coordination architecture.

### What confusion it removes

It removes the flattening move that treats communication as an after-the-fact wrapper around the "real" response.

### What distinction it introduces

It distinguishes between:

- executive command as an issuing centre
- public-information coordination as a structurally routed part of the response

### What current evidence supports it

- public-information coordination receives `25 / 30` coded events
- it remains active across escalation, implementation thickening, easing preparation, and de-escalation
- the current bounded null framing that reduces it to downstream messaging is weak against the ledger

### What would still need to be true before this becomes the single public payoff sentence

- the same pattern should survive a fuller ledger
- the pattern should survive the current estimator and sensitivity expansion
- the pattern should remain visible under at least one external comparison

## Candidate 2

### Candidate statement

SUF separates one crisis case into different coordination forms across windows instead of treating the whole response as one undifferentiated block.

### What confusion it removes

It removes the idea that a case either "worked" or "failed" in one uniform way from beginning to end.

### What distinction it introduces

It distinguishes:

- a thin setup window
- a dense perturbation window
- a coherent but lower-acuity transition window

### What current evidence supports it

- Comparator A, the main interval, and Comparator B already separate in event density, issuing-centre concentration, and lag pattern
- the main interval now carries a visibly denser implementation layer than the flanking windows
- the current conjoint comparison can already differentiate windows by coordination form rather than only by date

### What would still need to be true before this becomes the single public payoff sentence

- window differences should survive estimator hardening
- those differences should remain under the expanded perturbation program
- at least one comparator route should preserve the usefulness of the windowed distinction

## Candidate 3

### Candidate statement

SUF makes implementation-side coordination legible by showing how welfare support, emergency powers, border handling, and enforcement continuity are routed together rather than disappearing beneath headline decisions.

### What confusion it removes

It removes the tendency to let top-level announcements stand in for the whole coordination object.

### What distinction it introduces

It distinguishes:

- decision-side events
- implementation-side thickening
- legal and operating-authority routing
- welfare and delivery support pathways

### What current evidence supports it

- the route now includes direct DPMC and Police sources for stranded people, managed-isolation welfare, emergency-powers guidance, review preparation, local-authority/CDEM support, guidance clarification, checkpoints, and patrol continuity
- the main interval rose from a thinner headline chain to a `23`-event implementation-rich window without collapsing the earlier readout
- the route's strongest current empirical gain depends on this implementation-side visibility

### What would still need to be true before this becomes the single public payoff sentence

- the implementation-side layer should continue to deepen without breaking the current structure
- the estimator pass should preserve the gain instead of treating it as anecdotal thickening
- a comparator branch should show that this distinction is not unique to one curated case
