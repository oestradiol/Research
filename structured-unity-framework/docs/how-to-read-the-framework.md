# How to Read the Framework

**Full file map:** `docs/INDEX.md`

## 1. Who this document is for

Readers who want a clear path through the package without overload: newcomers, theory-first readers, route-first readers, and atlas-first readers.

## 2. Four reader paths

### Newcomer path

Goal: orientation without committing to every detail.

1. `README.md`
2. `docs/how-to-read-the-framework.md`
3. `docs/framework-overview.md`
4. `docs/claims-and-boundaries.md`
5. `framework/framework-interface.md`
6. skim `docs/glossary.md` as needed

### Theory-first path

Goal: understand the layered stack and boundary discipline.

1. `README.md`
2. `docs/framework-overview.md`
3. `docs/claims-and-boundaries.md`
4. `framework/framework-interface.md`
5. `framework/structural-phenomenology.md`
6. `framework/informational-awareness-framework.md`
7. `framework/unity-dynamics-framework.md`
8. `framework/research-program.md`
9. `framework/literature-guide.md`

### Demonstrated-route path

Goal: see the active case package, then ground it in theory.

1. `README.md`
2. `docs/claims-and-boundaries.md`
3. `applications/README.md`
4. `applications/demonstrated-routes/README.md`
5. `applications/demonstrated-routes/states-and-societies/institutional-coordination-under-perturbation/new-zealand-pandemic-case.md`
6. `applications/demonstrated-routes/states-and-societies/institutional-coordination-under-perturbation/event-ledger-seed.md`, `.../estimator-implementation.md`, `.../first-pass-seed-readout.md`, and `.../first-pass-window-comparison.md`
7. `framework/framework-interface.md` and `framework/research-program.md`

### Atlas-first path

Goal: inspect the wider research map before returning to the core theory.

1. `README.md`
2. `applications/README.md`
3. `applications/research-map/README.md`
4. one cluster gateway of interest
5. one or two node files of interest
6. return to `docs/claims-and-boundaries.md`, `framework/framework-interface.md`, and `framework/research-program.md`

## 3. Recommended reading order

1. `README.md`
2. `docs/how-to-read-the-framework.md`
3. `docs/framework-overview.md`
4. `docs/claims-and-boundaries.md`
5. `framework/framework-interface.md`
6. remaining `framework/` files
7. `applications/` as needed
8. `references/` for the source registry, citation policy, and curated reading lists
9. `meta/` for policies

## 4. What to skip on first pass

- technical route detail if you only want the conceptual structure: skim `framework/research-program.md`
- reference layer: start with `references/source-registry.md` if you need exact public source metadata, then `references/minimal-reading-list.md` or `references/standard-reading-list.md` for guided reading
- meta files: optional until you edit or extend the package

## 5. What not to assume too early

- that one demonstrated route equals the whole framework
- that `unity-like` implies strong consciousness claims
- that metrics are identical with experience
- that atlas breadth means SUF is already validated everywhere
- that `v1.0.0` means universal proof or publication-final closure

## 6. How the layers relate

- **Structural Phenomenology:** epistemic ground
- **Informational Awareness Framework:** structural and generative modeling layer
- **Unity Dynamics Framework:** integration, coherence, latency, and unity-like organization at explicit scales and windows
- **Framework Interface:** bridge and scope-control layer

Downstream in this package: **Research Program** (methods), **Literature Guide** (citation routing), **Applications Atlas** (demonstrated route plus wider atlas).

## 7. How to read the applications layer

Treat demonstrated routes as bounded uses of the framework and research-map nodes as scoped next-step sketches. Both are useful; neither should be mistaken for universal proof.

## 8. How to use the status surfaces

- use `docs/project-status.md` for the current package posture
- use `ROADMAP.md` for future-facing direction
- use `docs/pending-inventory.md` for open and deferred items

## 9. Assisted reading and accessibility support

This package is also meant to work with human assistants and reasoning agents acting in support roles.

If a reader wants guided navigation, paced reading, glossary help, or accessibility-oriented mediation, start with `../using-this-research-with-human-assistants-and-reasoning-agents.md` and keep the original file open while summarizing or explaining it.
